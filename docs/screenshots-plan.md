## Visual Style Guidelines

### Caption Styling

- **Font**: San Francisco (system font)
- **Size**: 32-48pt (iPhone), 48-64pt (iPad)
- **Weight**: Medium or Semibold (500-600)
- **Color**: White with 60% opacity background
- **Position**: Bottom 1/4 of screenshot
- **Alignment**: Center
- **Line height**: 1.2-1.4

### Device Frames

**Optional but recommended**:
- Use device frame for professional appearance
- Frame should be subtle (no glare, reflections)
- Match device type (iPhone SE, iPhone 15, etc.)

### Color Consistency

- Maintain consistent color palette across all screenshots
- Primary: Indigo (#6366F1)
- Secondary: Pink (#EC4899)
- Accent: Violet (#8B5CF6)

## Capture Workflow

### Per-Device Process

1. **Select Simulator**
   ```bash
   xcrun simctl boot "iPhone SE (3rd generation)"
   open -a Simulator
   ```

2. **Launch App**
   - Build and run in Xcode
   - Navigate to screenshot scene

3. **Prepare Scene**
   - Set up test data
   - Navigate to correct screen
   - Wait for animations to complete

4. **Capture Screenshot**
   - Press `Cmd+Shift+S` in Simulator
   - Or use: `xcrun simctl io booted screenshot <filename>.png`

5. **Verify Quality**
   - Check resolution matches device spec
   - Ensure no simulator chrome visible
   - Verify caption area is clean

6. **Process Image**
   - Run through processing script
   - Add caption overlay
   - Add device frame (optional)
   - Optimize file size

7. **Final Review**
   - Check against QA criteria
   - Verify consistent style
   - Confirm file naming

## Processing Pipeline

### Automated Processing

Use `scripts/process_screenshots.py`:

```bash
python3 scripts/process_screenshots.py \
  --device iphone_se \
  --input screenshots_raw/iphone_se/ \
  --output fastlane/screenshots/iphone_se/ \
  --caption-bottom \
  --frame-device
```

### Manual Processing Steps

1. **Resize** to exact device resolution
2. **Add caption** with semi-transparent background
3. **Add device frame** (optional)
4. **Optimize** with optipng
5. **Verify** meets App Store specs

## Quality Assurance

### Screenshot Validation Checklist

- [ ] Resolution matches device specification exactly
- [ ] No simulator chrome or status bar artifacts
- [ ] Caption is legible and centered
- [ ] Glass morphism effect is visible
- [ ] App UI is not obscured by captions
- [ ] Color accuracy maintained
- [ ] File size < 5 MB per screenshot
- [ ] File naming convention followed
- [ ] PNG format, lossless
- [ ] No pixelation or artifacts

### Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| Resolution wrong | Captured with wrong device | Re-capture with correct simulator |
| Caption illegible | Font too small or low contrast | Increase font size or background opacity |
| Simulator chrome visible | Captured full window | Use Simulator's screenshot feature (Cmd+S) |
| Glass effect not visible | Card not in correct state | Ensure card is flipped and depth effect active |
| Text cut off | Caption too low | Adjust caption positioning |
